# Методология: что считаем и откуда

Geo-Intelligence строит 3D-разбор района вокруг точки или внутри произвольной зоны. Один запрос `GET /api/scan?lat&lon&radius&label` (точка, зона ±radius м, по умолчанию 400) либо `GET /api/scan?polygon=lat,lon;lat,lon;…&label` (своя зона, 3–100 вершин) собирает всё сразу (здания, дороги, источники активности и все маски параллельно) и возвращает единый payload. Ниже — источники и формулы каждого компонента.

Режим зоны: центр и охват сцены берутся из bbox полигона (охват = максимальная полусторона ×1.15, минимум 250 м, максимум 1200 м — иначе 400). Данные загружаются по bbox, но значения масок вне полигона обнуляются (point-in-polygon по центрам ячеек), поэтому min/avg/max в инспекторе считаются только по зоне. Здания, дороги и активность вокруг зоны остаются как контекст. У сплошных масок (сейсмика, воздух — одно значение на весь район) число не зависит от формы зоны, обрезается только отрисовка.

Главный принцип: **не подделываем числа**. Каждый слой — либо реальные данные, либо модель, явно подписанная как модель прямо в интерфейсе.

## Здания

| | |
|---|---|
| Источники | **Overture Maps** (= OSM ∪ Google Open Buildings ∪ Microsoft ML footprints) и **OSM/Overpass** |
| Код | `lib/overture.ts`, `lib/scan.ts` |

Оба источника запрашиваются параллельно, берётся **более полный набор** (по числу зданий). В Европе обычно выигрывает OSM, в Global South — Overture (пример: Дели — 4335 зданий против 393 в OSM). Overture читается DuckDB-запросом (`bin/duckdb`) напрямую из публичного S3-бакета `overturemaps-us-west-2` (GeoParquet, релиз зафиксирован в `lib/overture.ts`), фильтр по bbox. Высота здания: `height` → `num_floors × 3 м` → дефолт 10 м.

## Дороги и ж/д

OSM/Overpass: все `highway`-линии + `railway=rail` в bbox. Ширина отрисовки — по классу дороги. Это же — входные данные шумовой модели.

## Источники активности

OSM/Overpass, 4 категории (`lib/activity-sources.ts`):

- **ночная жизнь** — nightclub, bar, pub
- **крупный ритейл** — mall, building=retail, marketplace
- **площадки и досуг** — stadium, cinema, theatre, events_venue
- **хабы и стройка** — bus_station, станции/метро/трамвай, construction, industrial

У каждого типа — качественный радиус влияния (кольцо на карте). Это **не дБ** и не входит в маски — отдельный слой с тумблами по категориям.

## Маска «Шум · Lden»

| | |
|---|---|
| Тип | **Модель** (не замер) |
| Входные данные | Классы дорог и ж/д из OSM |
| Код | `lib/noise-mask.ts`, `lib/noise-model.ts` |
| Сетка | 96×96 |

Каждому классу дороги присвоена референсная эмиссия на 10 м (motorway 80 дБ … service 50 дБ; ж/д 75 дБ). Дорога — линейный источник: спад **−10·log₁₀(d/10)** (−3 дБ на удвоение расстояния + лёгкое поглощение). В точке берётся энергетическая сумма ближайших источников по классам. Шкала цвета: 45 дБ (тихо) … 75+ дБ (громко); прозрачность растёт с громкостью (`t^1.6`).

**Ограничения:** не учитывает трафик по времени суток, экранирование зданиями, покрытие дороги. Не видит источники, которых нет в OSM (см. слой активности как частичную компенсацию). Путь апгрейда — официальные END-контуры EEA.

## Маска «Качество воздуха» (AQI)

| | |
|---|---|
| Тип | **Реальные данные** (модельный реанализ) |
| Источник | Open-Meteo Air Quality API → **CAMS** (Copernicus/ECMWF) |
| Код | `lib/air.ts` |
| Свежесть | Текущий час; кэш 1 час |

Берём European AQI (0–100) + PM2.5 + NO₂ текущего часа. CAMS — атмосферная модель, ассимилирующая спутники (Sentinel-5P) и наземные станции. Разрешение ~11 км ≫ тайла, поэтому поле **намеренно однородное** (ambient-тинт): внутрирайонной вариации в источнике нет, и мы её не выдумываем.

## Маска «Риск разлива рек»

| | |
|---|---|
| Тип | **Модель** (HAND-lite), не официальная карта затоплений |
| Рельеф | **Copernicus DEM GLO-30** (~30 м) — напрямую из публичного S3 (COG, range-чтение через `geotiff`), без квот |
| Вода | OSM/Overpass в радиусе **2.5×** от тайла |
| Код | `lib/flood.ts`, `lib/dem.ts` |
| Сетка | 48×48 |

Риск ячейки = высота над ближайшей значимой водой (HAND):

```
hand   = max(0, высота_ячейки − высота_ближайшей_воды)
риск   = clamp(1 − hand / 10 м) × 100%
если отрезок «ячейка → ближайшая вода» пересекает защитное сооружение:
риск  ×= 0.35   (инженерный дисконт)
```

**Инженерная защита:** из OSM берутся дамбы/валы/противопаводковые стены (`man_made=dyke|embankment|levee|flood_wall`, `barrier=flood_wall|flood_barrier`, `waterway=dam`) и насосные станции. Ячейка «за сооружением» (линия до её ближайшей воды пересекает вал) получает дисконт ×0.35 — вода должна преодолеть защиту. Дисконт эвристический и **защита не абсолютна** (Валенсия-2024); насосные станции считаются и показываются, но риск не снижают (их зона действия неизвестна). Если сооружения не замаплены в OSM — дисконта нет, о чём написано в примечании маски.

«Значимая вода»: линии `waterway=river|stream|canal|drain` — всегда; полигоны `natural=water` — только от **10 га** (пруды, фонтаны, бассейны игнорируются, включая теги water=pond/fountain/basin/…). Если значимой воды нет и в расширенном радиусе — риск честно ~0.

**Устойчивость получения воды** (строим до последнего): Overpass основное зеркало → 3 резервных зеркала (osm.ch, maps.mail.ru, kumi.systems; по одной попытке на зеркало — ретрай это следующее зеркало) → фолбэк на **Overture theme=base/water с S3** (те же фильтры по subtype/площади) → Redis-кэш. Пометка «⚠ данные недоступны» возможна только при одновременном отказе всех путей — отказ не маскируется под безопасность.

**Ограничения:** GLO-30 — DSM (поверхность: крыши завышают высоту ячейки → риск в плотной застройке может занижаться); нет гидрологии (дренаж, насосы, дамбы, осадки — Джакартские насосы и отвод Турии в Валенсии модель не знает); вода — только замапленная в OSM.

**Валидация на реальных городах:** Джакарта ~77% (тонущая пойма, 13 рек), Валенсия ~55% (модель подсветила старое русло Турии — линию наводнения 1957 года), Мадрид ~36% (плато; риск только коридорами вдоль Мансанареса и каналов).

## Маска «Прогноз наводнений» (Q100 · WRI)

| | |
|---|---|
| Тип | **Официальная глобальная модель** (peer-reviewed) |
| Источник | WRI Aqueduct Floods (GLOFRIS, Deltares/Utrecht) — глобальный растр с публичного S3 |
| Код | `lib/official-flood.ts` |
| Сетка | 48×48 (источник ~1 км) |

Глубина затопления (см) для сценария «раз в 100 лет»: **max(речной, прибрежный)** компонент; прибрежный — вариант **с учётом проседания грунта** (wtsub). Одна модель на всю планету — сравнима между любыми странами. Читается range-запросами из глобальных GeoTIFF (~2-3 с на тайл), кэш 30 дней. Сухо = прозрачно; любое затопление видимо (альфа-пол).

Роль слоя — **перекрёстная проверка**: наша рельефная «Риск разлива рек» отвечает «куда пойдёт вода по рельефу», Q100 — «что говорит официальная гидрологическая модель о 100-летнем паводке». Совпали — уверенность; разошлись — сигнал (инженерная защита, плювиальный риск вне модели, или грубость 1-км сетки).

**Ограничения:** разрешение ~1 км (на тайле это несколько пикселей — поле блочное); ливневые подтопления не входят (см. маску «Ливневое подтопление»); модель без учёта локальной защиты (undefended).

**Сценарий 2050:** тумблер под легендой переключает маску на климатический сценарий (RCP 8.5, модель NorESM1-M, 2050 год) из того же источника WRI. Значения могут быть и выше, и ниже сегодняшних — климат меняет осадки в обе стороны.

## Маска «Сейсмика»

| | |
|---|---|
| Тип | **Официальная глобальная модель** |
| Источник | GEM Global Seismic Hazard Map v2023 (Zenodo) |
| Код | `lib/seismic.ts` |

Пиковое ускорение грунта (PGA, % g) с вероятностью превышения 10% за 50 лет (период повторяемости ~475 лет), скальное основание. На масштабе района значение ~однородно → ambient-тинт (как воздух). Растр скачивается один раз (~33 МБ) и кэшируется локально; значения — в Redis (90 дней). **Лицензия CC BY-NC-SA** — для коммерческого использования нужна лицензия GEM (указано в подсказке маски). Валидация: Тбилиси 29 %g > Джакарта 23 > Валенсия 7 > Мадрид 2.

## Маска «Оползни»

| | |
|---|---|
| Тип | **Официальная глобальная модель** |
| Источник | NASA LHASA — карта восприимчивости к оползням (~1 км) |
| Код | `lib/landslide.ts` |
| Сетка | 48×48 |

Восприимчивость склонов (уклон, геология, разломы, дороги, вырубка леса), классы 1–5 показаны как 0–100%. Восприимчивость ≠ прогноз события. Растр ~33 МБ скачивается один раз, значения кэшируются (90 дней).

## Слой «Опасные соседи» (в активности) и ЛЭП

Категория источников активности `hazard` (OSM/Overpass): АЗС (`amenity=fuel`), резервуары топлива/газа (`man_made=storage_tank`), аэродромы (`aeroway=aerodrome`), электроподстанции (`power=substation`) — маркеры с зонами внимания. **Линии электропередачи** (`way[power=line]`) — отдельный переключаемый слой (красные линии) в секции «Слои». Это факторы восприятия/цены и страховые маркеры, не физическая модель риска.

## Маска «Ливневое подтопление»

| | |
|---|---|
| Тип | **Модель** (заполнение депрессий) |
| Рельеф | Copernicus DEM GLO-30 (тот же, что у затопляемости) |
| Код | `lib/pluvial.ts` |
| Сетка | 48×48 |

Классический алгоритм заполнения понижений (priority-flood): вода «наливается» в рельеф, глубина застоя = уровень заполнения − высота ячейки. Показывает, **где встанет вода при сильном ливне, если дренаж не справился**. Сухо/сток = прозрачно; лужи от 1 см видимы, шкала до 1 м.

**Ограничения:** ливневая канализация не учитывается (модель — сценарий её отказа); DSM-рельеф — понижения глубже 3 м отбрасываются как артефакты застройки (дворы среди высоток); ёмкость понижений не привязана к конкретному объёму осадков.

## Геокодинг

Поиск адреса: **Google Places API (New)** — серверные роуты `/api/suggest` (подсказки) и `/api/place` (координаты выбранного места); ключ не покидает сервер (`NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`/`GOOGLE_MAPS_API_KEY` в env, читается в рантайме). В Google Console должна быть включена **Places API (New)** и разрешена для ключа. Фолбэков нет. `/api/geocode` (Nominatim) — служебный эндпоинт; CLI геокодит через Nominatim напрямую.

## Кэширование

Всё в **Redis** (`docker compose up -d`, контейнер `geo-redis`, префикс `geo:`), TTL нативный:

| Данные | TTL |
|---|---|
| HTTP-ответы (Overpass, Nominatim) | 7 дней |
| Воздух (CAMS) | **1 час** |
| Overture-застройка | 30 дней |
| DEM-сетки высот | 30 дней |
| Q100-маска (WRI) | 30 дней |

При недоступном Redis приложение работает без кэша (graceful degrade). Внешние хосты защищены per-host rate-limit-очередью (`lib/http.ts`).

## Первый скан района

Холодный скан занимает до ~70 с — почти целиком это скан Overture-parquet по bbox в S3. Повторные сканы того же места — секунды (всё из Redis).

## CLI

`npm run scan -- "адрес" [--radius 500] [--out payload.json]` — тот же конвейер `computeScan`, что и у `/api/scan`, из терминала; `--out` пишет полный payload в JSON. Координаты: `--lat=… --lon=…` (отрицательные значения — через `=`).
